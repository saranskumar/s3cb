# S4 Command Center: Feature & UI Map

The S4 Command Center is an **execution-first** study blueprint designed for high-performance exam preparation. It prioritizes "What to do right now" over passive analytics.

## 1. Core Philosophy: Execution-First
- **Focus Mode**: The dashboard identifies the single most important task and the "Output Peak" (highest intensity block) for the day.
- **Minimal Distraction**: Management tools (Syllabus) are separated from action tools (Checklist).
- **Claymorphism UI**: A tactile, 3D aesthetic designed to make the interface feel alive and interactive.

## 2. Feature Breakdown

### A. Workspaces (Plans)
- **Multi-Plan Setup**: Users can manage multiple isolated exam goals (e.g., S4 Finals, CAT Prep).
- **Template Library**: Instant cloning of professional study blueprints.

### B. Daily Mission (The Daily Mission)
- **High-Performance List**: A sorted view of today's tasks across the active plan.
- **Atomic Actions**:
    - **Complete**: Instant progress update.
    - **Push (Move)**: Reschedules task to Tomorrow.
    - **Skip**: Marks as omitted for the day.

### C. Syllabus Master (Infrastructure)
- **Subject Hub**: Associated specifically with the active Workspace.
- **Topic Outline**: Flattened list (no modules) for faster management.
- **Mastery Tracking**: Visual Indicators of done/mastered status.

### D. Analytics (Intelligence)
- **Power Stats**:
    - **Overall Progress**: Workspace-specific completion rates.
    - **Hours Invested**: Time tracking per work session.
    - **Vulnerability Matrix**: Direct list of "Weak Topics" across the workspace.

## 3. UI System: Claymorphism
| Element | Visual Style |
|:--- |:--- |
| **Cards** | High border-radius (24px+), inner shadows, and soft highlights. |
| **Buttons** | Recessed on click, soft 3D shadows on hover. |
| **Dashboard** | Dynamic gradients (`clay-card-indigo`) for peak intensity blocks. |
| **Typography** | Heavy black weights (`font-black`) for clear hierarchy. |

## 4. Onboarding & Migration
- **Workspace First**: Users aren't forced into a single plan. They start by entering or creating a Workspace.
- **Flattened Curriculum**: Immediate access to topics without module nesting.
