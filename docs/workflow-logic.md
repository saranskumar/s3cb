# Workflow Logic — S3 Comeback

This document explains the complex logic flows within the application.

## 1. Data Synchronization

### Initialization
1.  App loads and checks `VITE_APPS_SCRIPT_URL`.
2.  Fetches data from Google Sheets.
3.  Parses 2D arrays into objects with `rowIndex` metadata for future updates.

### Updating (The "Update-Fetch" Loop)
1.  User triggers an action (e.g., toggle checkbox).
2.  **Optimistic Update**: UI updates immediately in the local `allData` state.
3.  **Confetti**: If the action was marking a task as "Done", a confetti burst is triggered.
4.  **Backend Sync**: A POST request is sent to the Apps Script.
5.  **Failure Handling**: If the request fails (offline or server error), the payload is pushed to the `offlineQueue` (LocalStorage).

### Reconnection Sync
1.  The app uses `useOnlineStatus` hook to track connectivity.
2.  When switching from offline to online:
    -   Loops through `offlineQueue`.
    -   Processes each action sequentially.
    -   Once cleared, triggers a full `fetchData()` to reconcile state with the server.

## 2. Reminders and Notifications

### Hourly Reminders
-   Controlled by the `hourlyReminders` state (`s3_reminders` in LocalStorage).
-   When enabled, a `setInterval` runs every 3600000ms (1 hour).
-   Randomly selects a motivational nudge from a predefined list and shows a Web Notification.

### Pomodoro Timer
-   A countdown timer (`25:00` default).
-   When it hits `00:00`, it sends a "Timer Done!" notification if permissions are granted.

## 3. PWA (Progressive Web App)
-   **Service Worker**: Generated by `vite-plugin-pwa` in `autoUpdate` mode.
-   **Caching**: Assets are cached for offline availability.
-   **Manifest**: Defines the app as `standalone` with a dark background color to ensure a smooth splash screen transition on mobile.
