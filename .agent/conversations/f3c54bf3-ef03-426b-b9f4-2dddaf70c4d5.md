## Objective
The primary goal was to eliminate UI lag and fix reactivity issues in the study tracker, particularly for task completions and rescheduling.

## Key Accomplishments
1.  **Optimistic UI Engine**: Implemented `onMutate` in `useDataMutation` to provide instant feedback for task actions (Complete, Skip, Move).
2.  **Atomic Sync**: Coupled `study_plan` task updates with `topics` table status changes.
3.  **Ghost Session Handling**: Fixed a critical issue where deleted users remained logged in by adding a `supabase.auth.getUser()` server-side verification check on app boot.
4.  **Advanced Rollover**: Added `moved_tomorrow` status allowing tasks to be pushed forward while remaining visible for "Undo" actions on the Today page.
5.  **Auto-Normalization**: Implemented a background cleanup that reverts "moved" tasks to "pending" once their scheduled date arrives.
